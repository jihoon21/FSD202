[ 009. 애플리케이션설계 키워드 체크 ]

1. 모듈의 개념
- 전체 프로그램의 기능 중 특정 기능을 처리할 수 있는 실행 코드를 의미하며 자체적으로 컴파일 가능하고 다른 프로그램에서 재사용이 가능하다.

2. 공통 모듈에 대한 명세 기법
- 공통 모듈에 대한 명세를 작성할 때에는 다음의 원칙을 지킨다.
(1) 정확성 : 해당 기능이 실제 시스템 구현 시 필요한지 여부를 알 수 있도록 정확하게 작성한다
(2) 명확성 : 해당 기능에 대해 일관되게 이해되고 한 가지로 해석될 수 있도록 작성한다
(3) 완전성 : 시스템이 구현될 떄 필요하고 요구되는 모든 것을 기술한다
(4) 일관성 : 공통 기능들 간에 상호 충돌이 없도록 작성
(5) 추적성 : 공통 기능에 대한 요구사항 출처와 관련 시스템 등의 유기적 관계에 대한 식별이 가능하도록 작성한다.

3. 재사용(Reuse)
- 개발 시간 및 비용 절감을 위하여 이미 검증된 기능을 파악하고 재구성하여 시스템에 응용하기 위해 적합하게 최적화시키는 작업이다.

4. 응집도의 유형
- 기능적/순차적/통신적/절차적/시간적/논리적/우연적

5. 결합도의 유형
- 자료/스탬프/제어/외부/공통/내용

6. 시스템 연동의 개념
(데이터 연동, 인터페이스(Interface) 연동,웹 서비스(Web Service) 연동
- 시스템 연동이란 서버와 테느워크를 활용하는 자동화 채계 등에서 동일한 기능을 수행하지 않는 단위 시스템 간에 접속을 통하여 업무(기능)를 수행하는 것을 의미한다.

7. 웹 서비스 구성 요소
- 웹 서비스 - SOAP/UDDI/WSDL

8. 미들웨어 솔루션의 개념
- 클라이언트와 서버 간의 통신을 담당하는 시스템 소프트웨어 또는 컴퓨터와 컴퓨터의 연결을 담당하는 소프트웨어로서, 중간을 의미하는 미들과 소프트웨어를 의미하는 웨어의 합성어이다.

9. 웹서버와 웹 애플리케이션 서버
- 웹서버
웹브라우저의 요청을 받아 html파일이나 이미지,자바스크립트의 정적인 콘텐츠를 제공하며 대표적인 웹서버로 아파치가있다
- 웹 애플리케이션 서버
서버단에서 애플리케이션이 동작할 수 있는 환경을 제공하고, 안정적인 트랜잭셩 처리 및 관리, 다른 이기종 시스템 간의 애플리케이션 연동을 지원한다.웹서버와의 가장 큰 차이는 동적 서버 콘텐츠를 수행할 수 있는 기능이다.

10. 재해 복구 시스템의 복구 수준별 유형
- 미러사이트/핫사이트/웜사이트
